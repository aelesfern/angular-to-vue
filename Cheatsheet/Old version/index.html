Angular Vs Vue

Angular
Vue

Introducción

  • Ofrece una solución completa para la mayoría de las necesidades de una aplicación web
  • En su estructura y documentación presta mucha importancia a la estandarización
  • Está totalmente integrado con Typescript y Rxjs
  • Suele utilizarse Ngrx como librería para implementar arquitectura Redux
  • Su core se centra en la capa de presentación para que sea mas sencillo trabajar con el
  • El uso de Typescript es opcional
  • No esta integrado con Rxjs, aunque ofrece una librería para su integración (poco mantenida)
  • Ofrece un mejor rendimiento (datos no actualizados)
  • Suele utilizarse Vuex como librería para implementar arquitectura Redux

CLI

  • Esta pensado para ser extendido con plugins (@vue/cli-plugin-xxx) y no tocar webpack
  • No tiene comandos para crear nuevas piezas, como por ejemplo componentes

Componentes

import { Component, EventEmitter, OnChanges, Output, SimpleChanges } from '@angular/core';

@Component({
	selector: 'app-user-list', // Selector para uso en HTML
	templateUrl: './user-list.component.html',
	styleUrls: ['./user-list.component.scss'],
})
export class AppUserList implements OnChanges {
	users: any[] = [];
	@Output() userRemoved: EventEmitter = new EventEmitter(); // Salida de datos
	lastViewedUser: any; // Datos propios

	userQuantity() { // Datos calculados
	return this.users.length;
	}
	ngOnChanges(changes: SimpleChanges) { // Reacción a cambios
	changes.users && console.log(changes.users.currentValue.length);
	}
	removeUser(user: any) { // Método
	this.userRemoved.emit(user);
	}
}
  • Se centran en la lógica relacionada con la representación visual
  • Puede tener su CSS y HTML en archivos únicos o incluidos en el archivo .component
// Componente .js
export default {
	name: 'AppUserList', // Selector para uso en HTML

	}
}

import { Injectable } from "@angular/core";
import { HttpClient } from "@angular/common/http";

@Injectable({providedIn: 'root'})
export class UserService {
	private users: any[] = [];

	constructor(private httpService: HttpClient) { }

	getUsers() {
		return this.httpService.get(url).pipe(
			first(),
			tap(users => this.users = users)
		);
	}
}
  • Pensados para proveeder lógica específica a una funcionalidad no relacionada directamente con la representación visual
  • Estan dentro del sistema de inyección de dependencias, lo que permite crear código modular organizando jerarquicamente
  • Puesto que su lógica no debe ser específica a una vista por defecto son Singletons compartidos por la aplicación
  • Pueden crearse multiples instancias del mismo servicio para casos de uso poco habituales
let users = [];

export default {
	getUsers() {
		return fetch(url).then(newUsers => {
			users = newUsers;
			return users;
		});
	}
}
  • No existe una pieza definida como servicio
  • Usando VanillaJS puede crearse un análogo a los servicios singleton
  • Vue ofrece como alternativa para crear código reutilizable por múltiples componentes los Mixins

Mixins de Vue

// Mixin
var myMixin = {
	created: () => this.hello(),
	methods: {
		hello: () => console.log('hola desde el mixin!')
	}
}

// Componente que usa el mixin
var Component = Vue.extend({
	mixins: [myMixin]
})

var component = new Component() // => "hola desde el mixin!"
  • Son utilizados para abstraer una lógica de un componente y hacerla re-utilizable
  • Un mixin es una configuración de componente, parcial o no
  • Al usar un mixin, un componente integra su configuración como propia
  • Un componente puede utilizar múltiples mixins
  • Un mixin puede a su vez utilizar otros mixins.
  • Pueden registrarse mixins de forma global. Todos los componentes tendrán esé código, lo usen o no
  • Recomendación: utilizar con moderación, su uso puede dificultar la comprensión de una base de código de gran tamaño.
  • Recomendación: al diseñarlos sigue siempre el principio de responsabilidad única.

Directivas

import { Directive, ElementRef, TemplateRef, ViewContainerRef } from "@angular/core";

@Directive({ selector: '[encebollado]'})
export class EncebolladoDirective {
	constructor(private elementRef: ElementRef){}
	
	encebollado() {
	this.elementRef.nativeElement.innerHTML = `
${this.elementRef.nativeElement.innerHTML} con cebolla
`;
	}
}
  • Permiten añadir comportamientos nuevos a componentes y etiquetas html
  • @angular/core ofrece un set de directivas que cubren las necesidades más habituales (ngFor, ngIf, etc...)
Vue.directive('encebollado', { // Registra una directiva global llamada v-encebollado
	inserted: function(el) {
		el.nativeElement.innerHTML = `
${el.nativeElement.innerHTML} con cebolla
`;
	}
});
  • Son muy parecidas a las de Angular, aunque no hacen diferencia entre las de atributo y las estructurales
  • Pueden asociarse a ciertos eventos del componente o etiqueta usados

Routing

  • La configuración del routing puede dividirse en módulos
  • Pueden configurarse Guards asociados a rutas para controlar el acceso a las mismas
  • Ofrece una librería cuyo uso es similar a @angular/routing, para la que existe una documentación detallada
  • La configuración del routing se aglutina en un único archivo
  • Los Guards pueden definirse a nivel de ruta, y también a nivel de componente
  • Puede separarse código en diferentes archivos para carga vaga usando imports en la configuración del enrutado

HttpClient

  • Ayuda a tipar las respuestas, gestionar los errores, interceptar las llamadas y facilitar el testeo
  • No proporciona una interfaz para hacer peticiones http
  • Se sugiere el uso de la librería axios, para la que existe documentacion de ejemplo

Testing

  • Angular CLI pre-configura el entorno para el uso de Karma + Jasmine para Unit testing
  • Angular CLI pre-configura el entorno para el uso de Protractor para E2E testing
  • La creación de piezas nuevas con el Cli genera automaticamente scripts de testeo
  • Vue CLI permite elegir como entorno de unit testing Mocha + Chai o Jest
  • Vue CLI permite elegir como entorno de E2E testing Cypress, Nightwatch y WebdriverIO
  • Las principales librerías de herramientas para testeo son @testing-library/vue y @vue/test-utils
  • Se recomienda utilizas Jest para introducirse al testing en Vue
  • Existe una documentación específica para testing no-oficial
